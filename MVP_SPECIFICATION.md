# TheraType MVP (Minimum Viable Product) 명세서

**문서 목적**: 4주 내 구현 가능한 기능 정의 및 개발 계획

**원칙**: 과장 없이, 실현 가능한 것만 포함

---

## 📋 MVP 정의

### MVP란?

**Minimum Viable Product**: 핵심 가치를 검증할 수 있는 **최소한의 기능**을 가진 제품

**TheraType MVP의 목적**:
- ✅ 핵심 컨셉 (타이핑 + 정신건강) 작동 가능성 증명
- ✅ 사용자 수용성 및 만족도 확인
- ✅ 데이터 수집 시스템 검증
- ✅ 향후 개발 방향 결정을 위한 피드백 수집

**MVP가 아닌 것**:
- ❌ 완성된 제품
- ❌ 임상적으로 검증된 치료제
- ❌ 상용 서비스 가능한 수준

### 2. Insight Mode (자기인식 모드)

#### 2.1 기능 범위

**문장 쌍 개수**: 10개 (5개 카테고리 × 2개씩)

| 카테고리 | 문장 쌍 ID | 문장 A | 문장 B |
|---------|-----------|--------|--------|
| 자기인식 | SP1 | 나는 내 장점과 강점을 잘 알고 있다 | 나는 내 단점이 더 많이 눈에 띈다 |
| 자기인식 | SP2 | 나는 충분히 잘하고 있다고 느낀다 | 항상 더 잘해야 한다는 압박감이 있다 |
| 스트레스 대응 | SR1 | 어려움이 와도 결국 해결할 수 있다 | 문제가 생기면 감당하지 못할 것 같다 |
| 스트레스 대응 | SR2 | 스트레스를 받으면 휴식을 취한다 | 스트레스를 받으면 더 일에 몰두한다 |
| 사회적 에너지 | SE1 | 혼자 있는 시간이 나를 회복시킨다 | 사람들과 함께 있을 때 에너지를 얻는다 |
| 사회적 에너지 | SE2 | 깊은 대화를 나누는 소수의 친구 선호 | 다양한 사람들과 가볍게 교류하는 걸 즐김 |
| 감정 조절 | ER1 | 내가 왜 이런 감정을 느끼는지 이해한다 | 갑자기 감정이 올라와서 당황스럽다 |
| 감정 조절 | ER2 | 부정적 감정도 자연스럽게 받아들인다 | 부정적 감정을 느끼면 나쁜 사람 같다 |
| 미래 지향성 | FO1 | 미래에 좋은 일이 기다리고 있을 거라 믿는다 | 미래가 불확실하고 불안하다 |
| 미래 지향성 | FO2 | 어제보다 나은 내가 되고 있다 | 제자리걸음인 것 같아 답답하다 |

### 3. Therapy Mode (치료 모드)

#### 3.1 기능 범위

**문장 라이브러리**: 총 20개 문장

| 프로파일 | 문장 개수 | 난이도 |
|---------|----------|--------|
| 자존감 향상형 | 5개 | 초급 2, 중급 2, 고급 1 |
| 스트레스 관리형 | 5개 | 초급 2, 중급 2, 고급 1 |
| 감정조절형 | 5개 | 초급 2, 중급 2, 고급 1 |
| 동기부여형 | 5개 | 초급 2, 중급 2, 고급 1 |

**자존감 향상형 문장 예시**:
1. (초급) 나는 소중한 사람이다 (11자)
2. (초급) 나는 존중받을 자격이 있다 (14자)
3. (중급) 실수는 나를 정의하지 않는다 (16자)
4. (중급) 나는 충분히 노력하고 있으며 그것만으로도 가치 있다 (28자)
5. (고급) 타인의 기준이 아닌 나만의 기준으로 나를 평가한다 (28자)

*나머지 프로파일의 문장은 CONCEPT_DETAIL.md 참조*

#### 3.2 프로파일 매칭 로직 (간단 버전)

```javascript
// Insight Mode 결과 기반
function assignProfile(insightScores) {
  const lowestCategory = Object.keys(insightScores)
    .reduce((a, b) => insightScores[a] < insightScores[b] ? a : b);

  const profileMap = {
    'self_perception': 'self_esteem',      // 자존감 향상형
    'stress_response': 'stress_management', // 스트레스 관리형
    'emotion_regulation': 'emotion_control', // 감정조절형
    'future_orientation': 'motivation'      // 동기부여형
  };

  return profileMap[lowestCategory];
}
```

## ❌ 제외되는 기능 (Scope OUT)

### 명확히 하지 않을 것

1. **고급 AI/ML 개인화**
   - ❌ 딥러닝 기반 문장 추천
   - ❌ 실시간 감정 분석
   - → 대신: 규칙 기반 간단한 알고리즘

2. **소셜 기능**
   - ❌ 친구 추가
   - ❌ 리더보드
   - ❌ 커뮤니티 게시판
   - → 이유: 프라이버시 우선, MVP 범위 초과

3. **모바일 네이티브 앱**
   - ❌ iOS/Android 앱
   - → 대신: 반응형 웹 (모바일 브라우저에서 작동)

4. **복잡한 게이미피케이션**
   - ❌ 아바타, 가상 상점, 복잡한 포인트 시스템
   - → 대신: 간단한 배지, 스트릭만

5. **음성/비디오**
   - ❌ 음성 안내
   - ❌ 비디오 튜토리얼
   - → 대신: 텍스트, 이미지만

6. **타언어 지원**
   - ❌ 영어, 일본어 등
   - → 한국어만

7. **고급 통계 분석**
   - ❌ 예측 모델 (dropout prediction)
   - ❌ 복잡한 패턴 분석
   - → 대신: 기본 기술 통계만 (평균, 추이)

8. **외부 통합**
   - ❌ Google Calendar, Notion 연동
   - ❌ 웨어러블 기기 연동
   - → 이유: 시간 부족

9. **결제 시스템**
   - ❌ 유료 구독, 결제
   - → 완전 무료 (MVP 단계)

10. **의학적 주장**
    - ❌ "우울증 치료", "불안 완화" 등의 의학적 표현
    - → 대신: "스트레스 관리 지원", "기분 개선 도움" 등 온건한 표현

## 🛠️ 기술 스택

### 프론트엔드
- **Framework**: React 18+
- **Language**: JavaScript (또는 TypeScript, 선택적)
- **Styling**: CSS Modules 또는 Tailwind CSS
- **Routing**: React Router v6
- **Charts**: Chart.js 또는 Recharts
- **State Management**: React Context API (간단한 상태만 있으므로 Redux 불필요)

### 백엔드 / 인프라
- **Backend as a Service**: Firebase
  - **Authentication**: 이메일/비밀번호
  - **Database**: Cloud Firestore
  - **Hosting**: Firebase Hosting
- **CI/CD**: GitHub Actions (선택적)

### 개발 도구
- **Version Control**: Git + GitHub
- **Package Manager**: npm 또는 yarn
- **Code Editor**: VS Code
- **Design**: Figma (와이어프레임) 또는 종이 스케치

- **Hosting**: Firebase Hosting (무료 티어)

## 👥 사용자 테스트 계획

### 참가자 모집

**목표 인원**: 10-20명

**모집 방법**:
- 대학 커뮤니티 (에브리타임, 학과 단톡방)
- SNS (Instagram, Facebook 친구 공유)
- 주변 지인 직접 요청

**모집 문구 예시**:
```
🔥 타자연습 + 정신건강 플랫폼 "TheraType" 테스터 모집!

- 기간: 2주
- 시간: 하루 15분
- 혜택: 타이핑 실력 향상 + 참가 기념품 (1만원 상당)

관심 있으신 분은 [링크] 클릭!
```

**참가 기준**:
- 19-45세
- 한글 타이핑 가능 (속도 무관)
- PC 또는 태블릿 보유
- 주 3회 이상 참여 가능

### 테스트 프로토콜

#### 1단계: 사전 설문 (5분)
**수집 정보**:
- 인구통계학 (나이대, 성별, 직업)
- 현재 타이핑 속도 (자가 평가: 느림/보통/빠름)
- 현재 기분/스트레스 수준 (1-5점)
- TheraType에 대한 기대

#### 2단계: 2주 사용 기간
**권장 사용 패턴**:
- Week 1, Day 1: Insight Mode 완료 (20분)
- Week 1, Day 2-7: Therapy Mode (매일 15분)
- Week 2, Day 1-7: Therapy Mode 계속 (매일 15분)
- Week 2, Day 7: Insight Mode 재평가 (선택적)

**모니터링**:
- 사용 데이터 실시간 확인 (대시보드)
- 중간 체크인 (1주 후): 간단한 피드백 수집

#### 3단계: 사후 설문 (10분)
**수집 정보**:
- 타이핑 속도 실제 측정 (TheraType 내 테스트)
- 기분/스트레스 수준 재측정 (1-5점)
- **System Usability Scale (SUS)**:
  - 10개 문항, 5점 척도
  - 사용성 평가 표준 도구
  - 예: "이 시스템을 자주 사용하고 싶다" (1=매우 아니다 ~ 5=매우 그렇다)
- 주관식 피드백:
  - 가장 좋았던 점
  - 개선이 필요한 점
  - 계속 사용하고 싶은지
  - 친구에게 추천하겠는지

### 데이터 분석 계획

#### 정량 분석
```
타이핑 속도 변화:
- 사전 평균: X WPM
- 사후 평균: Y WPM
- 변화: +Z WPM (paired t-test, p<0.05 목표이지만 필수 아님)

사용자 참여도:
- 평균 세션 수: N회/2주
- 평균 세션 길이: M분
- 완료율: K% (14일 중 몇 일 사용)

사용성 (SUS):
- 평균 점수: M (SD=X)
- 68점 이상: 양호
- 80점 이상: 우수

기분/스트레스 변화:
- 사전 평균: A점
- 사후 평균: B점
- (해석 주의: 샘플 작아 일반화 어려움)
```

#### 정성 분석
- 자유 응답 텍스트 분석
- 주요 테마 코딩:
  - 긍정적 피드백 (예: "재미있었다", "도움이 됐다")
  - 부정적 피드백 (예: "지루했다", "문장이 부족")
  - 개선 제안 (예: "모바일 앱 필요", "음악 추가")
- 대표 인용구 선정 (발표 자료 활용)
